## Smart Contract Development Rules

When developing smart contracts, please refer to the following rules file:

**Reference**: [`.cursor/rules/solidity.mdc`](.cursor/rules/solidity.mdc)

This file contains the following content related to Solidity and smart contract security:

- **General Rules**: Code style, writing principles, etc.
- **Solidity Best Practices**: Function visibility, naming conventions, security patterns, etc.
- **Testing and Quality Assurance**: Testing strategies, security audits, etc.
- **Performance Optimization**: Gas optimization, storage layout, etc.
- **Development Workflow**: Foundry usage, CI/CD pipelines, etc.
- **Documentation**: Code documentation guidelines
- **UI and Frontend Development**: Next.js, React, TypeScript usage guides
- **Web3 and Blockchain Integration**: ethers.js, viem, Web3 library usage

Please follow the above rules when developing smart contracts to write safe and efficient code.
